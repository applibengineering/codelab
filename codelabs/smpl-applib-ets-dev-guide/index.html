Build basic HarmonyOS App in eTS

In this codelab, you'll learn how to build and run your first HarmonyOS app in the eTS programming language.

What you must know

This codelab is written for programmers and assumes that you know the Typescript programming language. If you are an experienced programmer and adept at reading code, you will likely be able to follow this codelab, even if you don't have much experience with Typescript.

What you'll learn

In this codelab, you're going to get familiar with

  • How to use DevEco Studio to build your app.
  • How to run your app in the emulator.
  • How to add interactive components
  • How to display a second screen when a button component is pressed

Use DevEco Studio and Java to write HarmonyOS apps

You write HarmonyOS apps in the eTS programming language using an IDE called DevEco Studio. Based on JetBrains' IntelliJ IDEA software, DevEco Studio is an IDE designed specifically for HarmonyOS development.

You can download DevEco Studio from the DevEco Studio page.

DevEco Studio provides a complete IDE, including an advanced code editor and app templates. It also contains tools for development, debugging, testing, and performance that make it faster and easier to develop apps. You can use DevEco Studio to test your apps with a large range of preconfigured emulators.

DevEco Studio is available for computers running Windows and for Macs running macOS. The NodeJS and NPM is bundled with DevEco Studio which are required for JS / eTS development

Note: DevEco Studio is continually being improved. For the latest information on system requirements and installation instructions, see the DevEco Studio download page..

  1. Navigate to the DevEco Studio download page and follow the instructions to download and install DevEco Studio.
  2. Accept the default configurations for all steps, and ensure that all components are selected for installation.
  3. After the install is complete, the setup wizard downloads and installs additional components, including the HarmonyOS SDK. Be patient, because this process might take some time, depending on your internet speed.
  4. When the installation completes, DevEco Studio starts, and you are ready to create your first project.
  1. In this step, you will create a new HarmonyOS project for your first app. This simple app displays the string "Hello World" on the screen of a HarmonyOS virtual device.

What you'll learn

  • How to create a project in DevEco Studio.
  • How to run your app on the emulator.

Step 1: Create a new project

  1. Open DevEco Studio.
  2. In the Welcome to DevEco Studio dialog, click Create HarmonyOS Project

  1. Select Empty Ability and click Next.

  1. Give your application a name such as ETSSampleApp.
  2. Choose Project Type as Application.
  3. Select language as eTS
  4. Set Compatible API Version.
  5. Select the Device Type.
  6. Click Finish

After these steps, DevEco Studio:

  • Creates a folder for your DevEco Studio project called ETSSampleApp.
  • Builds your project (this may take a few moments). DevEco Studio uses Gradle as its build system. You can follow the build progress at the bottom of the DevEco Studio window.
  • Opens the code editor showing your project.

Step 2: Explore the project structure and layout

Based on you selecting the Empty Ability template for your project, DevEco Studio has set up a number of files for you. You can look at the hierarchy of the files for your app in multiple ways, one is in Project view. Project view shows your files and folders structured in a way that is convenient for working with a HarmonyOS project. (This does not always match the file hierarchy! To see the file hierarchy, choose the Project files view by clicking (1).)

  1. Double-click the ETSSampleApp (3) folder to expand the hierarchy of app files. (See (3) in the screenshot.)
  2. If you click Project (2), you can hide or show the Project view. You might need to select View > Tool Windows to see this option.
  3. The current Project view selection (1) is Project > Project or Project > Project Files.

In the Project > Project Files view you see the entry folder.

  1. Expand the entry folder.
  2. You see src > main folder, expand the main folder, you will see : ets, resources and config.json

This folder contains config.json. This file describes all the components of your HarmonyOS app and is read by the HarmonyOS runtime system when your app is executed.

  1. Expand the ets folder. All your eTS language files are organized here. The ets folder contains:
  1. The app.ets file manages global application logic and application lifecycle.
  2. The pages directory stores all component pages.
  3. The common directory stores public resource files, such as media resources, custom components, and common methods.
  4. The resources directory stores resource configuration files, for example, for multi-resolution loading.
  1. Expand the java folder. com.example.etssampleapp: This folder contains the Java source code files for your app.
  2. You also see src > ohosTest folder, expand the ohosTest folder. This folder is where you would put your framework tests, which are tests that run on a HarmonyOS device.

Step 3: Run your app on emulator

In this task, you will use the DevEco Device Manager to create a virtual device (or emulator).

  1. In DevEco Studio, select Tools > Device Manager.

2. After Logging in using developer ID, you will be able to access a list of available Virtual Devices. (Please note that you would need minimum API version 7 to run an ETS application)

3. Select any device and click run icon to start the device.

4. In DevEco Studio, select Run > Run ‘entry' or click the Run icon in the toolbar. The icon will change when your app is already running.

5.. Once your app builds and the emulator is ready, DevEco Studio uploads the app to the emulator and runs it. You should see your app as shown in the following screenshot.

Each visible page in a Harmony ETS app is collection of components that defines the user interface for the Page. eTS files allows you to build pages based on components and events. Pages build using ets have advanced capabilities such as logic control, databinding, event binding, loop rendering and conditional rendering.

eTS framework have prebuilt decorators with special meanings for decorating classes, structures, methods, and variables that define component behavior in app. For example @Entry, @Component and @State we use in following sample.

Step 1: Create the UI

  1. UI for eTS application are found in .ets files on the left side in Project panel. (entry > src > main > ets > pages)
  2. EmptyAbility Template project by default creates "Hello World" sample code, and you see index.ets default page is added to it.

We will modify "Hello World" screen to one shown below and on click of a button page slides to second page.

Step 2: UI Elements

  1. Harmony supports multiple built-in UI components like Text, Button, Input etc. These components are arranged together by adding Component containers like Flex, Row, Column etc. to build a page.
  2. "Flex" component container is used to arrange a Text and Navigator button in columns. As shown below.

  1. A component is tiny reusable UI module, @Component decorator is used to declare a component.
  2. @Entry decorator which is added just above @Component Decorator in index.ets page, declares that, when the page gets loaded, the entry component is the one which will get executed first.
  3. All Application's business logic code are placed in ets folder and Resource folder for app resource like strings, image, color, float values, Rawfiles.
  4. For example String resources are added string.json, and same is accessed in app with resource key like ‘$r(app.string.hello_world), where hello_world key to access string.

Step 3: Extend a component

We can extend Built-in component to support additional feature with @Extend Decorator. Here we created Text component as fancy Text component by extending with fancy method. And used across page to give similar Textual representation

Step 4: Configuring the Device Type

  1. Add "Phone" for smartphones and "tv" for smart TVs to the "deviceType" field in the config.json file.

Step 5: Defining Events

  1. Events mainly include gesture events for touchscreen devices such as wearables and key events for smart TVs.
  2. Examine the onclick event attribute defined for the button and on click of this event console log get printed.
