Build basic HarmonyOS App in JS

In this codelab, you'll learn how to build and run your first HarmonyOS app in the JS programming language.

What you must know

This codelab is written for programmers and assumes that you know the JS programming language. If you are an experienced programmer and adept at reading code, you will likely be able to follow this codelab, even if you don't have much experience with JS.

What you'll learn

In this codelab, you're going to get familiar with

  • How to use DevEco Studio to build your app.
  • How to run your app in the emulator.
  • How to add interactive components
  • How to display a second screen when a button component is pressed

Use DevEco Studio and Java to write HarmonyOS apps

You write HarmonyOS apps in the JS programming language using an IDE called DevEco Studio. Based on JetBrains' IntelliJ IDEA software, DevEco Studio is an IDE designed specifically for HarmonyOS development.

You can download DevEco Studio from the DevEco Studio page.

DevEco Studio provides a complete IDE, including an advanced code editor and app templates. It also contains tools for development, debugging, testing, and performance that make it faster and easier to develop apps. You can use DevEco Studio to test your apps with a large range of preconfigured emulators.

DevEco Studio is available for computers running Windows and for Macs running macOS. The NodeJS and NPM is bundled with DevEco Studio which are required for JS development

Note: DevEco Studio is continually being improved. For the latest information on system requirements and installation instructions, see the DevEco Studio download page..

  1. Navigate to the DevEco Studio download page and follow the instructions to download and install DevEco Studio.
  2. Accept the default configurations for all steps, and ensure that all components are selected for installation.
  3. After the install is complete, the setup wizard downloads and installs additional components, including the HarmonyOS SDK. Be patient, because this process might take some time, depending on your internet speed.
  4. When the installation completes, DevEco Studio starts, and you are ready to create your first project.

In this step, you will create a new HarmonyOS project for your first app. This simple app displays the string "Hello World" on the screen of a HarmonyOS virtual device.

What you'll learn

  • How to create a project in DevEco Studio.
  • How to run your app on the emulator.

Step 1: Create a new project

  1. Open DevEco Studio.
  2. In the Welcome to DevEco Studio dialog, click Create HarmonyOS Project

  1. Select Empty Ability and click Next.

  1. Give your application a name such as JSSampleApp.
  2. Choose Project Type as Application.
  3. Select language as JS
  4. Set Compatible API Version.
  5. Select the Device Type.
  6. Click Finish

.

After these steps, DevEco Studio:

  • Creates a folder for your DevEco Studio project called JSSampleApp.
  • Builds your project (this may take a few moments). DevEco Studio uses Gradle as its build system. You can follow the build progress at the bottom of the DevEco Studio window.
  • Opens the code editor showing your project.

Step 2: Explore the project structure and layout

Based on you selecting the Empty Ability template for your project, DevEco Studio has set up a number of files for you. You can look at the hierarchy of the files for your app in multiple ways, one is in Project view. Project view shows your files and folders structured in a way that is convenient for working with a HarmonyOS project. (This does not always match the file hierarchy! To see the file hierarchy, choose the Project files view by clicking (1).)

  1. Double-click the JSSampleApp(3) folder to expand the hierarchy of app files. (See (3) in the screenshot.)
  2. If you click Project (2), you can hide or show the Project view. You might need to select View > Tool Windows to see this option.
  3. The current Project view selection (1) is Project > Project or Project > Project Files.

In the Project > Project Files view you see the entry folder.

  1. Expand the entry folder.
  2. You see src > main folder, expand the main folder, you will see : js, resources and config.json

This folder contains config.json. This file describes all the components of your HarmonyOS app and is read by the HarmonyOS runtime system when your app is executed.

  1. Expand the js folder. All your JS language files are organized here. The js folder contains:
  1. The app.js file manages global JavaScript logics and application lifecycle.
  2. The pages directory stores all component pages.
  3. The common directory stores public resource files, such as media resources, custom components, and .js files.
  4. The i18n directory is used to configure resources in different languages, for example, UI strings and image paths.
  5. The resources directory stores resource configuration files, for example, for multi-resolution loading.
  1. Expand the java folder. com.example.jssampleapp: This folder contains the Java source code files for your app.
  2. You also see src > ohosTest folder, expand the ohosTest folder. This folder is where you would put your framework tests, which are tests that run on a HarmonyOS device.

Step 3: Run your app on emulator

In this task, you will use the DevEco Device Manager to create a virtual device (or emulator).

  1. In DevEco Studio, select Tools > Device Manager.

  1. After Logging in using developer ID, you will be able to access a list of available Virtual Devices.

  1. Select any one of device and click run icon to start the device.

  1. In DevEco Studio, select Run > Run ‘entry' or click the Run icon in the toolbar. The icon will change when your app is already running.
  2. Once your app builds and the emulator is ready, DevEco Studio uploads the app to the emulator and runs it. You should see your app as shown in the following screenshot.

Each visible Page in a Harmony JS app has a layout that defines the user interface for the Page which is written using HML.

The HarmonyOS Markup Language (HML) is an HTML-like language that allows you to build pages based on components and events. Pages built using HML have advanced capabilities such as logic control, data binding, event binding, loop rendering, and conditional rendering.

Events are bound to components through 'on' or '@'. When a component triggers an event, the corresponding event processing function in the .js file is executed.

Cascading Style Sheets (CSS) is a language used to describe the HML page structure. All HML components have default styles. You can customize styles for these components using CSS to design various pages.

What you'll learn

  • Exploring HML and css files.
  • How to add attributes, how to add string resources.

Step 1: Create the UI

  1. Find and open the HML file on the left side in the Project panel. (entry > src > main > js > pages > index)
  2. Double-click the HML file and its respective css and js file will be in the same folder.
  3. Right to Layout Editor is the Previewer (2), which shows a visual representation of what the contents of the file will look like when compiled into an HarmonyOS app. You can view the visual representation, the HML/CSS/JS code, or both.

Step 2: UI Elements

  1. In the HML code, notice that the root element is marked to .container style whose details can be found in .css file and contains 2 child elements, Text and Button.
  2. Examine properties in Text element for the added string and notice that the string value definition and value can be found in entry > src > main > js > default > i18n > <language>.json file and referred in HML file using $t function.

  1. You should be able to change the existing String property value to another String. You will also be able to add String resources for other component text attribute.
  2. You should be able to change the style and properties of the Text element using .title in CSS
  3. Notice that, instead of defining style attributes in CSS file, can also be done directly using style attribute to the element with in HML. (button element in above screenshot)
  4. Notice the type attribute of the button element which is set to capsule to display capsule-like button. Can try out with different other types like circle, arc etc.,

Step 3: Configuring the Device Type

  1. Add "Phone" for smartphones and "tv" for smart TVs to the "deviceType" field in the config.json file.
  2. In order to adjust or define the styles based on deviceType, can use

@media screen and (device-type: <type of device>) and enclose the styles within.

Step 4: Defining Events

  1. Events mainly include gesture events for touchscreen devices such as wearables and key events for smart TVs.
  2. Examine the onclick event attribute defined for the button and on click of this event the launch function in the javascript (.js file) file would be executed.

You have added Text to your app's main screen along with button and with no other functionality. In this task, you will make your button respond when the user clicks on it.

On click of Button Component you will navigate to Second Screen.

What you'll learn

  • How to implement Second Page.
  • How to Navigate between Pages.

Many applications consist of more than one page. We need to link these pages through page routing to implement redirection.

The page router finds the target page based on the Page URI.

Step 1: Create the Second Page

  1. In the Project window of DevEcoStudio, choose entry > src > main > js > default. Right-click the pages folder and choose New > JS Page from the shortcut menu to create the next page.

Step 2: Navigate to next page

  1. On click of the button, Call router.push() to add the page URI to the route stack, that is, to navigate users to the page specified by the URI. Note to import the router module before calling the router method.
  2. Please note that router.back() can be used to navigate users back to index page
