Build Basic HarmonyOS App in Java

In this codelab, you'll learn how to build and run your first HarmonyOS app in the Java programming language.

What you must know already

This codelab is written for programmers and assumes that you know the Java programming language. If you are an experienced programmer and adept at reading code, you will likely be able to follow this codelab, even if you don't have much experience with Java.

What you'll learn

  • How to use DevEco Studio to build your app.
  • How to run your app in the emulator.
  • How to add interactive text components.
  • How to display a second screen when a text component is pressed.

Use DevEco Studio and Java to write HarmonyOS apps

You write HarmonyOS apps in the Java programming language using an IDE called DevEco Studio. Based on JetBrains' IntelliJ IDEA software, DevEco Studio is an IDE designed specifically for HarmonyOS development.

You can download DevEco Studio from the DevEco Studio page.

DevEco Studio provides a complete IDE, including an advanced code editor and app templates. It also contains tools for development, debugging, testing, and performance that make it faster and easier to develop apps. You can use DevEco Studio to test your apps with a large range of preconfigured emulators.

Note: DevEco Studio is continually being improved. For the latest information on system requirements and installation instructions, see the DevEco Studio download page.

DevEco Studio is available for computers running Windows and for Macs running macOS. The OpenJDK (Java Development Kit) is bundled with DevEco Studio.

The installation is similar for all platforms. Any differences are noted below.

  1. Navigate to the DevEco Studio download page and follow the instructions to download and install DevEco Studio.
  2. Accept the default configurations for all steps, and ensure that all components are selected for installation.
  3. After the install is complete, the setup wizard downloads and installs additional components, including the HarmonyOS SDK. Be patient, because this process might take some time, depending on your internet speed.
  4. When the installation completes, DevEco Studio starts, and you are ready to create your first project.

In this step, you will create a new HarmonyOS project for your first app. This simple app displays the string "Hello World" on the screen of a HarmonyOS virtual device.

Step 1: Create a new project

  1. Open DevEco Studio.
  2. In the Welcome to DevEco Studio dialog, click Create HarmonyOS Project.

  1. Select Empty Ability(Java) and click Next.

  1. Give your application a name such as Basic App.
  2. Choose Project Type as Application.
  3. Set Compatible API Version.
  4. Select the Device Type.
  5. Click Finish.

After these steps, DevEco Studio:

  • Creates a folder for your DevEco Studio project called MyFirstApp.
  • Builds your project (this may take a few moments). DevEco Studio uses Gradle as its build system. You can follow the build progress at the bottom of the DevEco Studio window.
  • Opens the code editor showing your project.

Step 2: Explore the project structure and layout

Based on you selecting the Empty Ability(Java) template for your project, DevEcoStudio has set up a number of files for you. You can look at the hierarchy of the files for your app in multiple ways, one is in Project view. Project view shows your files and folders structured in a way that is convenient for working with a HarmonyOS project. (This does not always match the file hierarchy! To see the file hierarchy, choose the Project files view by clicking (1).)

  1. Double-click the BasicApp(3) folder to expand the hierarchy of app files. (See (3) in the screenshot.)
  2. If you click Project (2), you can hide or show the Project view. You might need to select View > Tool Windows to see this option.
  3. The current Project view selection (1) is Project > Project.

In the Project > Project view you see the entry folder.

  1. Expand the entry folder.
  2. You see src > main folder, expand the main folder, you will see : java, resources and config.json

This folder contains config.json. This file describes all the components of your HarmonyOS app and is read by the HarmonyOS runtime system when your app is executed.

  1. Expand the java folder. All your Java language files are organized here. The java folder contains:

com.example.basicapp: This folder contains the Java source code files for your app.

  1. Expand the resources folder. This folder contains all the resources for your app, including images, layout files, strings, icons, and styling. It includes these subfolders:

media: All your app's images will be stored in this folder.

layout: This folder contains the UI layout files for your Ability & AbilitySlices.

graphic: This folder contains the background elements and selector files.

element: This folder contains resources, such as strings and colors, used in your app.

  1. You also see src > ohosTest folder, expand the ohosTest folder

com.example.basicapp (ohosTest): This folder is where you would put your framework tests, which are tests that run on an HarmonyOS device.

com.example.basicapp (test): This folder is where you would put your unit tests. Unit tests don't need a HarmonyOS device to run. It starts out with a skeleton unit test file.

Step 3: Run your app on your new emulator

In this task, you will use the DevEco Device Manager to create a virtual device (or emulator).

  1. In DevEco Studio, select Tools > Device Manager.

  1. After Logging in using developer ID, you will be able to access a list of available Virtual Devices.

  1. Select any one of device and click run icon to start the device.

  1. In DevEco Studio, select Run > Run ‘entry' or click the Run icon in the toolbar. The icon will change when your app is already running.
  2. Once your app builds and the emulator is ready, DevEco Studio uploads the app to the emulator and runs it. You should see your app as shown in the following screenshot.

Generally, each screen in your Harmony app is associated with one or more AbilitySlices. The single screen displaying "Navigate to Second Screen!" is created by one AbilitySlice, called MainAbilitySlice. This was generated for you when you created your new project. Each visible AbilitySlice in an Harmony app has a layout that defines the user interface for the AbilitySlice. DevEco Studio has a layout editor where you can create and define layouts.

Layouts are defined in XML. The layout editor lets you define and modify your layout either by coding XML or by using the interactive visual editor.

Every element in a layout is a Component. In this task, you will learn how to change the properties of components.

What you'll learn

  • How to use the layout editor.
  • How to set property values, how to add string resources.

Step 1: Open the layout editor

  1. Find and open the layout folder (entry > src > main > resources > layout) on the left side in the Project panel(1).
  2. Double-click ability_main.xml.
  3. In the center is the Layout Editor(2) , where you can add XML code for Components and Layouts and their attributes.
  4. Right to Layout Editor is the Previewer (3), which shows a visual representation of what the contents of the file will look like when compiled into an HarmonyOS app. You can view the visual representation, the XML code, or both.
  5. Top Right corner of Previewer is a Component Tree (4) showing the views currently in this file, and how they are arranged in relation to each other.
  6. Bottom right corner of the Previewer is Attributes Panel (5),You'll learn about that later.

Step 2: Explore Component Tree

  1. In ability_main.xml, look at the Component Tree.

  1. This panel shows the component hierarchy in your layout, that is, how the views are arranged in relation to each other.
  2. If it's not showing you can open Component Tree by clicking Inspector from Previewer Window.

Step 3: Explore view hierarchies

  1. In the Component Tree, notice that the root of the component hierarchy is a DirectionalLayout component. Every layout must have a root component that contains all the other components. The root component is always a component container, which is a component that contains other components. A DirectionalLayout is one example of a component container.
  2. Notice that the DirectionalLayout contains a Text component, called text_helloworld.

  1. In the XML code, notice that the root element is <DirectionalLayout >. The root element contains a <Text> element.
<DirectionalLayout
  ... >

   <Text
      ...  />

</DirectionalLayout>

Step 4: Change property values, add String resources

  1. In the code editor, examine the properties in the Text element.
<Text
   ohos:id="$+id:text_helloworld"
   ohos:height="match_content"
   ohos:width="match_content"
   ohos:text="$string:mainability_HelloWorld"
   ...
   />
  1. Click on the string in the text property, and you'll notice it refers to a string resource, mainability_HelloWorld.
  2. Right-click on the property and click Go To > Declaration or Usages.

resources > base > element > strings.json opens with the string highlighted.

{
 "name": "mainability_HelloWorld",
 "value": "Hello World"
}
  1. You should be able to change the existing String property value to another String. You will also be able to add String resources for other component text attribute.

Your app's layout is now basically complete, but its appearance can be improved with a few small changes.

Step 1: Add new color resources

  1. In resources > base > element > colors.json , Add a new color named screenBackground. Use the value #D3D3D3, which is a lighter shade in the gray palette
  2. Add a new color named textBackground. Use the value #FFFFFF, which is white.
{
 "color": [
   {
     "name": "screenBackground",
     "value" : "#D3D3D3"
   },
   {
     "name": "textBackground",
     "value" : "#FFFFFFFF"
   }
 ]
}

Step 2: Add background elements for the Text and DirectionalLayout

  1. 1In resources > base > graphic, create screen_background.xml and set this xml as background element for DirectionalLayout.
<?xml version="1.0" encoding="UTF-8" ?>
<shape xmlns:ohos="http://schemas.huawei.com/res/ohos"
      ohos:shape="rectangle">
   <solid
       ohos:color="$color:screenBackground"/>
</shape>
  1. Create text_background.xml and set this xml as background element for the Text component.
<?xml version="1.0" encoding="UTF-8" ?>
<shape xmlns:ohos="http://schemas.huawei.com/res/ohos"
      ohos:shape="rectangle">
   <solid
       ohos:color="$color:textBackground"/>
</shape>
  1. In the ability_main.xml layout , add background_element attribute for DirectionalLayout and Text.
<?xml version="1.0" encoding="utf-8"?>
<DirectionalLayout
   xmlns:ohos="http://schemas.huawei.com/res/ohos"
   ohos:height="match_parent"
   ohos:width="match_parent"
   ohos:background_element="$color:screenBackground"
   ...>

   <Text
       ohos:id="$+id:text_helloworld"
       ohos:height="match_content"
       ohos:width="match_content"
       ohos:background_element="$graphic:text_background"
       ...
       />

</DirectionalLayout>

Step 3: Update the appearance of the Text component

1. Decrease the text size of the Text component to 18fp.

2. Add Text Color attribute and change the text from "Hello World" to "Navigate to Second Screen!"

3. Add Padding to make text spaced from margins.

    <Text
       ...
       ohos:text="$string:mainability_navigate"
       ohos:padding="10vp"
       ohos:text_size="18fp"
       ohos:text_color="$color:textColor"
       />

Step 1: Implement the Second slice

  1. Right click src > com.example.basicapp.slice and create New -> Java Class named SecondAbilitySlice.java extending AbilitySlice,

  1. Right click resources > base > layout and create ability_second_slice.xml to create UI components via xml code,

<DirectionalLayout

Step 2: Making Text component Interactive

  1. Get back to First screen, Open MainAbilitySlice.java (src > java > com.example.basicapp.slice > MainAbilitySlice).
  2. This class has a method called onStart(). This method executes when the AbilitySlice starts.
  3. The id for a component helps you identify that component distinctly from other components. Using the findComponentByID() method, your code can find the text_helloworld using its id, ResourceTable.Id_text_helloworld.
  4. Take a look at onStart(). It sets up a click listener for the text_helloworld.
Text textHelloWorld = (Text) findComponentById(ResourceTable.Id_text_helloworld);
textHelloWorld.setClickedListener(new Component.ClickedListener() {
   @Override
   public void onClick(Component component) {
       present(new SecondAbilitySlice(), new Intent());
   }
});

Here is what this code does:

  • Use the findComponentByID() method with the id of the desired component as an argument, then set a click listener on that component.
  • In the body of the click listener, use an action, which in this case is for navigating to SecondAbilitySlice.
  1. Run your app. Press the Navigate to Second Screen! Text and watch it navigates to the Second Screen.
